## Strict naming convention

Use this format for every Markdown file:

```text
YYYY-MM-DD__short_lowercase_topic.md
```

Rules:

```text
- Date first: YYYY-MM-DD
- Separator: double underscore __
- Topic in lowercase
- Words separated by underscores
- No spaces
- No umlauts
- No special characters except underscores and hyphens in dates
- File extension: .md
```

Examples:

```text
2026-05-21__wallet_roles.md
2026-05-21__trust_model.md
2026-05-21__business_relevance_matrix.md
```

---

## Date rule

The date in the filename represents the latest meaningful content version.

When the content changes substantially, update the filename date or create a new dated note.

Small typo fixes do not require a new date.

Examples:

```text
2026-05-21__trust_model.md
2026-05-28__trust_model.md
```

or:

```text
2026-05-21__trust_model.md
2026-05-28__trust_model_update.md
```

Use a new dated file when the new content represents a new research step, changed understanding, or important revision.
